Common models & approximations (and why engineers use them)

  • Linearization: keep first-order terms in a small parameter ε; neglect O(ε²).
  • Steady-state: set ∂/∂t = 0 when transients have died out.
  • Symmetry reduction: reduce PDE dimensionality using invariances (e.g., cylindrical symmetry).

PART 2 — Strategy & Tips (Roadmap Only)

Pending: This roadmap depends on your specific problem (PDE/ODE/eigenproblem/etc.).

Roadmap template (5–12 steps max)

  1. Define the model: choose coordinates, state variables, domain, and assumptions.
  2. Write governing equation(s): in operator form (e.g., ∇·J=0, Δu=0, iħ∂ψ/∂t=Ĥψ).
  3. Nondimensionalize (if useful): pick scales, identify key dimensionless groups.
  4. Apply symmetry / reductions: reduce dimensionality or simplify the operator.
  5. Solve: separation of variables / integrating factor / Fourier / Green’s function / energy method as appropriate.
  6. Apply boundary/initial conditions: determine constants/eigenvalues.
  7. Compute target quantity: substitute back, simplify, and present result.
  8. Sanity checks: units, limiting cases, sign/direction logic.
Common mistakes (general)
  • Skipping boundary conditions until the end (then constants don’t match).
  • Forgetting vector directions (sign errors in ∇, ∇·, ∇×).
  • Mixing phasor and time-domain quantities without stating convention.
  • Using an approximation without stating what terms are dropped (order tracking).
